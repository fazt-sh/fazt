# Preline Component Library - Design System Documentation

> **Purpose:** This document provides comprehensive markup examples for converting a design system to Preline UI components. All markup is React-compatible and uses Tailwind CSS utility classes.

> **Source:** [Preline UI Documentation](https://preline.co/docs/index.html) | [GitHub Repository](https://github.com/htmlstreamofficial/preline)

> **Note:** Preline requires the Preline JavaScript plugin for interactive components (Modal, Dropdown, etc.). Include the plugin via CDN or npm installation.

## Design System Overview

This documentation maps your design system components to equivalent Preline UI components. Each section includes complete HTML markup that can be converted to React components by an LLM agent.

**Key Principles:**
- All components use Tailwind CSS utility classes
- Dark mode support included via `dark:` prefix classes
- Responsive design with breakpoint prefixes (`sm:`, `md:`, etc.)
- Accessibility attributes included (`aria-*`, `role`, etc.)

## Colors

Your design system colors mapped to Preline/Tailwind CSS classes:

| Design System Color | RGB Value | Preline/Tailwind Class | Usage |
|---------------------|-----------|------------------------|-------|
| Primary | rgb(255, 149, 0) | `bg-orange-500` / `text-orange-500` | Primary actions, links |
| Success | rgb(34, 197, 94) | `bg-green-500` / `text-green-500` | Success states, confirmations |
| Error | rgb(239, 68, 68) | `bg-red-500` / `text-red-500` | Error states, destructive actions |
| Info | rgb(59, 130, 246) | `bg-blue-500` / `text-blue-500` | Informational content |

**Note:** The exact RGB values provided do not match standard Tailwind color scales. The classes above use the closest semantic equivalents. For exact color matching, extend your Tailwind configuration:

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: 'rgb(255, 149, 0)',
        success: 'rgb(34, 197, 94)',
        error: 'rgb(239, 68, 68)',
        info: 'rgb(59, 130, 246)',
      }
    }
  }
}
```



## Installation and Setup

### CDN Installation

To use Preline components with interactive functionality, include the Preline JavaScript plugin:

```html
<!-- Include Preline JS from CDN -->
<script src="https://cdn.jsdelivr.net/npm/preline@2.5.0/dist/preline.js"></script>
```

### NPM Installation

```bash
npm install preline
```

Then import in your JavaScript:

```javascript
import 'preline';
```

### Tailwind CSS Configuration

Add Preline to your `tailwind.config.js`:

```javascript
module.exports = {
  content: [
    './node_modules/preline/dist/*.js',
    './src/**/*.{html,js,jsx,ts,tsx}',
  ],
  plugins: [
    require('preline/plugin'),
  ],
}
```

## React Conversion Notes

When converting these HTML examples to React components:

1. **Replace `class` with `className`**
2. **Replace `for` with `htmlFor`** (in label elements)
3. **Convert data attributes:** `data-hs-overlay` becomes `data-hs-overlay` (no change needed)
4. **Self-closing tags:** Ensure all self-closing tags end with `/>`
5. **Event handlers:** Convert to camelCase (e.g., `onclick` â†’ `onClick`)
6. **Boolean attributes:** Use explicit boolean values (e.g., `disabled={true}`)

### Example React Conversion

**HTML:**
```html
<button type="button" class="py-3 px-4" disabled>
  Button
</button>
```

**React:**
```jsx
<button type="button" className="py-3 px-4" disabled={true}>
  Button
</button>
```
